# auto-learn - 自動知識蓄積

**このスキルはClaude自身が自動的に呼び出す。ユーザーの指示は不要。**

## 発動トリガー（1つでも該当すれば即座に実行）

| # | トリガー | 具体例 |
|---|----------|--------|
| 1 | バグ修正で根本原因が非自明だった | 祖先のpadding-inlineがspace-betweenのgapを消していた |
| 2 | Figma→CSS変換で落とし穴があった | mix-blend-modeがREST APIエクスポートに反映されない |
| 3 | ワークフローの手順不足で問題を見落とした | クイック視覚チェックがなく画像重複に気づけなかった |
| 4 | ビルド・環境で予想外の動作があった | Gulpのwatch中にSCSS変更が反映されない |
| 5 | 同じ種類のミスを2回以上した | letter-spacingのem計算を間違えた等 |
| 6 | MCPの挙動が想定外だった | get_design_contextがSVGを分解して返した |
| 7 | 調整を3回以上繰り返した | font-sizeの値を何度も修正した |

## 発動しない場合

- 単純なタイポ修正
- ユーザーの指示による単純な値変更
- 既にagent.mdに同じ内容が記録されている

## 実行手順

### 1. agent.md を読み込む

`docs/agent.md` を読み、既存のセクション構造と内容を把握する。

### 2. 重複チェック

今回の発見に関連するキーワードで agent.md を検索する。
既に同じパターンが記録されていれば、新規追加ではなく **既存エントリに実例を追加** する。

### 3. エントリを作成

以下のフォーマットで作成する：

```markdown
### 落とし穴: [簡潔なタイトル]

[何が起きたか（具体的な状況）]

**実例**: [実際のコード・値・セレクター名を含む再現可能な記述]

**対策**:
1. [防止手順1]
2. [防止手順2]
```

**必須ルール**:
- 抽象的な教訓は書かない。具体的なセレクター名、CSS値、計算式を含める
- 「〜に注意する」「〜を確認する」だけでは不十分。何を、どの値で、どう確認するかを書く
- 直前の作業で使った実際の値・クラス名・ノードIDを含める

### 4. 適切なセクションに追記

agent.md の既存セクションで最も近いものの末尾に追記する。
合わない場合は新しいセクションを作成する。

### 5. 行数チェック

- 500行超: 「agent.md が{N}行です」と警告
- 1000行超: PIPELINE.md の分割ルールに従い自動分割を実行

### 6. 報告

ユーザーに **1行で** 報告する。例:
```
agent.mdに「absoluteアイコンのpadding-top計算ルール」を追記しました（現在{N}行）
```

## 重要: このスキルの存在意義

PIPELINE.md の絶対ルール8と同じ内容だが、**スキルとしてコンテキストに常時ロードされる**ことで、
長いセッション中でも忘れずに agent.md への書き込みが実行される。

作業中に上記トリガーに該当する事象が起きたら、**作業を中断せずに**このスキルを実行すること。
修正作業の直後に agent.md を更新し、その後通常の作業に戻る。
