# /learn - agent.md への知識記録

ユーザーが `/learn` で呼び出したとき、以下の手順で `docs/agent.md` に知識を追記する。

## 手順

### 1. 引数の解析

`$ARGUMENTS` にユーザーが記録したい知識の概要が渡される。
引数がない場合は「何を記録しますか？」と聞く。

### 2. agent.md の現在の状態を確認

1. `docs/agent.md` を読み込む
2. 行数を確認する（500行超なら警告、1000行超なら分割を実行）
3. 既存のセクション一覧を把握する

### 3. 重複チェック

- 引数のキーワードで `docs/agent.md` を Grep 検索する
- 類似のエントリが既にあれば、新規追加ではなく **既存エントリに実例を追加** する

### 4. エントリのフォーマット

以下のフォーマットで追記内容を作成する：

```markdown
### 落とし穴: [簡潔なタイトル]

[何が起きたか（具体的な状況）]

**実例**: [実際のコード・値・セレクター名を含む再現可能な記述]

**対策**:
1. [防止手順1]
2. [防止手順2]
```

#### ルール
- **抽象的な教訓は書かない**。必ず具体的な実例（セレクター名、CSS値、計算式など）を含める
- ユーザーの引数が抽象的な場合は、直前の作業コンテキストから具体例を抽出する
- 「パターン」「改善事項」の場合は「落とし穴」の代わりに適切なプレフィックスを使う

### 5. 適切なセクションに追記

- agent.md の既存セクション構造を見て、最も適切なセクションの末尾に追記する
- 既存セクションに合わない場合は、新しいセクション（番号付き `## N. タイトル`）を追加する

### 6. 行数チェックと報告

- 追記後の行数を確認する
- 500行超: 「agent.md が{N}行です。1000行超で自動分割します」と警告
- 1000行超: PIPELINE.md の自動分割ルールに従って即座に分割を実行する
- ユーザーに「agent.md に『〇〇』を追記しました（現在{N}行）」と1行で報告する
