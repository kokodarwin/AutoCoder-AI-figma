# 工程3: 実装計画（自動発動）

**レイヤー分析（工程2）完了後に自動発動。計画書を作成してからコーディングに進む。**

## 手順

### 1. パターンの選定

`docs/patterns/README.md` → 該当パターンファイルを参照。

| # | パターン | 主な用途 |
|---|---------|---------|
| 01 | レイアウトコンテナ | inner + セクション余白 |
| 02 | ヘッダー・ナビ | 固定/非固定ヘッダー |
| 03 | ハンバーガー・ドロワー | SPメニュー |
| 04 | セクションタイトル | 共通見出し |
| 05 | カードグリッド | 2/3/4列一覧 |
| 06 | メディアブロック | 画像+テキスト横並び |
| 07 | CTA・コンタクト | 行動喚起 |
| 08 | ボタン | リンクボタン |
| 09 | フッター | 複数列フッター |
| 10 | フロー・ステップ | 手順表示 |
| 11 | 重なり・並び替え | negative margin（absolute不使用） |
| 12 | 背景装飾 | 波・半円・三角（pseudo-element） |

選定基準: レイヤー分析結果の構造とパターンの「Figmaでの特徴」を照合。複数組み合わせ可。

### 2. HTML構造の設計

- セクション全体のネスト図
- 各要素のHTMLタグ（div, section, figure, span, h2, h3等）
- クラス名（`.p-{section}__{element}` 形式）
- セマンティックHTML要素の配置

### 3. SCSS方針の決定

- レイアウト手法（flex / grid / 通常フロー）
- レスポンシブ戦略（ブレークポイントごとの変更内容）
- 背景の実装方法（CSS / 1枚画像書き出し）
- 特殊プロパティ（transform, mix-blend-mode等）

### 4. 画像リストの確定

- ファイル名の命名
- 取得方法（REST API / アセットURL）
- 形式（SVG / PNG / JPG）
- 保存先パス

### 5. リスク箇所の対処方針

- CSS再現困難な装飾 → 画像書き出し
- テキストの部分スタイル → span分離
- レスポンシブで崩れやすい箇所 → ブレークポイント戦略
- `position: absolute` が必要に見える箇所 → 代替手法

### 6. 計画書の保存

`docs/mcp-log/{page}/{section}-plan.md` に保存。
