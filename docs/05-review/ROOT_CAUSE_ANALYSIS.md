# 根本原因分析（RCA）フレームワーク

## 概要

視覚検証（工程6）でpixelmatch不合格となった場合、差分を表面的に直すのではなく、構造化された原因分析を行う。

---

## 差分カテゴリ分類

差分画像（diff.png）の赤い部分を以下のカテゴリに分類する:

| カテゴリ | 説明 | 例 |
|---------|------|-----|
| A. 位置ズレ | 要素全体がX/Y方向にずれている | セクション全体が10px下にずれている |
| B. サイズ差異 | 要素のwidth/heightが異なる | カードの幅が30px広い |
| C. 余白差異 | margin/padding/gapが異なる | カード間のgapが大きすぎる |
| D. テキスト差異 | フォントサイズ、行間、文字間、改行位置 | letter-spacingが未設定 |
| E. 色差異 | 背景色、テキスト色、ボーダー色 | 背景色の変数が間違っている |
| F. 形状差異 | border-radius、shadow、border | 角丸が設定されていない |
| G. レンダリング差異 | フォントレンダリング、アンチエイリアス | コードで修正不可（許容） |

---

## 「なぜ」3回掘り下げ

差分を特定したら、3回「なぜ」を繰り返して根本原因に到達する。

### 例: カード間のgapが大きすぎる

```
差分: カード間の余白がFigmaより20px大きい

なぜ1: gap: rem(60) と指定しているが、Figmaでは40px
→ Figma値の読み間違い

なぜ2: MCPログのgap値を確認せずに実装した
→ 工程5のセルフレビューでDカテゴリ（Figma値の正確性）をスキップした

なぜ3: セルフレビューチェックリストを部分的にしか実行しなかった
→ プロセスの問題

根本原因: セルフレビューの不徹底
修正: Figma値を再確認してgapを修正 + 他の値も再確認
```

### 例: 背景色が微妙に異なる

```
差分: セクション背景が微妙に明るい

なぜ1: $bg-light を使用しているが、Figmaの背景色が $bg-light と異なる
→ 変数の値が不正確

なぜ2: _variables.scss の $bg-light の定義値がFigmaと一致していない
→ 工程0（デザインシステム構築）での変数定義ミス

なぜ3: Figmaの最新カラーが反映されていない
→ 変数定義後にFigma側が更新された可能性

根本原因: 変数値の不一致
修正: Figmaの現在の背景色を確認し、_variables.scssの値を更新
```

---

## よくある根本原因パターン

| # | 根本原因 | 発生頻度 | 対策 |
|---|---------|---------|------|
| 1 | **Figma値の読み間違い** | 高 | MCPログと実装の突合を徹底（工程5-D） |
| 2 | **変数の値が不正確** | 中 | _variables.scssとFigmaの値を再確認 |
| 3 | **レイアウト手法の選択ミス** | 中 | パターンライブラリの推奨手法に立ち返る |
| 4 | **レスポンシブ切り替えの不足** | 中 | 全ブレークポイントでの表示確認 |
| 5 | **画像取得の不備** | 低〜中 | 計画書の画像リストと実ファイルを照合 |
| 6 | **複合背景のCSS実装失敗** | 低 | 3層以上は迷わず1枚画像書き出し |
| 7 | **フォントレンダリング差異** | 高 | コードで修正不可 → 許容する |

---

## カテゴリ別 修正アプローチ

### A. 位置ズレ

1. 親要素のレイアウト（flex/grid）を確認
2. margin-top / margin-bottom の値を確認
3. padding の値を確認
4. 該当しない場合 → 上位要素のサイズ差異が伝播していないか確認

### B. サイズ差異

1. width / height / max-width の値をFigmaと照合
2. padding が含まれていないか確認（box-sizing）
3. flex-shrink / flex-grow の設定を確認
4. aspect-ratio の値を確認

### C. 余白差異

1. gap / margin / padding の値をFigmaと照合
2. ショートハンドで指定した場合、方向の指定ミスがないか確認
3. padding-block / padding-inline の使い方を確認

### D. テキスト差異

1. font-size をFigmaと照合
2. line-height をFigmaと照合
3. letter-spacing を計算し直す（px ÷ font-size = em）
4. font-weight をFigmaと照合
5. font-family が正しい変数か確認

### E. 色差異

1. 使用している変数の定義値を確認
2. Figmaの色コードと変数値を照合
3. opacity の設定を確認
4. rgba() の値を確認

### F. 形状差異

1. border-radius をFigmaと照合
2. box-shadow をFigmaと照合（x, y, blur, spread, color）
3. border の幅・色・スタイルを確認

### G. レンダリング差異

- **コードで修正不可** → 許容する
- フォントレンダリング（ヒンティング、サブピクセル）
- アンチエイリアスの差異
- OS/ブラウザ固有の描画差異

---

## 使い方

1. 工程6で差分画像を分析する際、このフレームワークを参照する
2. 差分を上記カテゴリに分類する
3. 「なぜ」3回で根本原因を特定する
4. カテゴリ別の修正アプローチに従って修正する
5. 修正後、再検証する
