# MCPログ保存ルール

## 概要

Figma MCP で取得したデザイン情報を保存するディレクトリです。
取得した情報を保存しておくことで、後続の品質チェックやコードとの整合性確認時に、MCP に再接続することなく参照できます。

---

## 保存先構成

```
docs/mcp-log/
├── README.md           # このファイル
├── {page}/             # ページごとのディレクトリ
│   ├── design-context.md   # デザイン情報
│   └── variables.md        # カラー・フォント変数
└── common/             # 共通コンポーネント
    └── design-context.md
```

---

## 保存すべき情報

### 1. デザインコンテキスト（design-context.md）

`get_design_context` で取得した情報を保存します。

**必須項目（1つでも欠けたらコーディング禁止）:**
- レイアウト情報（gap, padding, margin - **px値で正確に**）
- サイズ（width, height - 小数点含む）
- フォント情報（fontFamily, fontSize, fontWeight, lineHeight, letterSpacing）
- 色情報（**HEXコードで正確に。「白」「青系」等の自然言語記述は厳禁**）
- テキスト内容
- **グラデーション（角度deg + 各色HEX + 停止位置%）**
- **box-shadow（完全な値）**
- **border-radius（px値）**
- **opacity（小数値）**
- **要素座標（top/left/right/bottom - 親要素基準で記録）**
- **テキストweight範囲（どの文字列にどのweightが適用されるか正確に）**

**保存例:**

```markdown
# トップページ デザインコンテキスト

## 取得日時
2026-01-29 08:50

## セクション: Hero

### レイアウト
- display: flex
- gap: 40px
- padding: 80px 0

### 背景
- グラデーション: linear-gradient(103.67deg, #3e9ccd 0.15%, #1e7aaa 98.4%)
- border-radius: 6px
- box-shadow: 0 0 30px rgba(6, 56, 82, 0.08)

### タイポグラフィ
| 要素 | font-size | font-weight | line-height | letter-spacing | 適用範囲 |
|------|-----------|-------------|-------------|----------------|----------|
| 見出し | 48px | 700 | 1.4 | 2.4px | 全体 |
| 本文 | 16px | 400 | 1.8 | 0px | 「など、」以降 |
| 本文(強調) | 16px | 700 | 1.8 | 0px | 「〜強化したい」まで |

### カラー
| 要素 | カラーコード | 備考 |
|------|-------------|------|
| 見出し | #042275 | ※「navy」等の名称ではなくHEX必須 |
| 本文 | #232323 | |
| ボタン背景 | #eef6fa | ※「白」ではなく正確なHEX |

### 要素配置
| 要素 | top | left | width | height | opacity |
|------|-----|------|-------|--------|---------|
| 背景ロゴ | 130px | 246px | 748px | 219.587px | 0.1 |
```

### 2. 変数情報（variables.md）

`get_variable_defs` で取得したカラー・フォント変数を保存します。

**保存例:**

```markdown
# カラー・フォント変数

## 取得日時
2026-01-29 08:50

## カラー変数
| 変数名 | カラーコード |
|--------|-------------|
| --navy | #042275 |
| --black | #232323 |
| --white | #FFFFFF |
| --gray-bg | #F5F5F5 |

## フォント変数
| 変数名 | フォント |
|--------|----------|
| --base-font | "Yu Gothic", "游ゴシック", sans-serif |
| --second-font | "Lexend", sans-serif |
```

## 保存ルール

### 1. タイミング

- **コーディング開始前**に必ず保存
- セクションごとに取得した場合は都度追記

### 2. ファイル命名

```
docs/mcp-log/{ページ名}/design-context.md
docs/mcp-log/{ページ名}/variables.md
```

**ページ名の例:**
- `top` - トップページ
- `about` - Aboutページ
- `contact` - お問い合わせページ
- `common` - 共通コンポーネント（ヘッダー、フッター等）

### 3. 更新時

- 情報が更新された場合は、取得日時を更新して上書き
- 古い情報は履歴として残す必要はない（最新のみ保持）

## 画像取得のベストプラクティス

### デフォルト動作

**画像取得の指示がない限り、画像は取得しません。**

- `get_design_context` を使用する際も、`dirForAssetWrites` は指定しない
- デザイン情報（フォント、カラー、レイアウト、テキスト内容など）のみを取得

### 画像が必要な場合

**詳細**: [01-assets/ASSETS.md](../01-assets/ASSETS.md) を参照

1. 明示的に「○○の画像を取得してください」と指示
2. REST API でダウンロード
3. `src/img/{ページ名}/` に保存
4. 命名規則: `{セクション名}{数字}.{拡張子}` (例: hero1.jpg)

## MCPログの活用シーン

### 1. 品質チェック時

[07-qa/QA.md](../07-qa/QA.md) のチェック項目で参照：
- タイポグラフィの照合
- カラーの照合
- 画像パスの確認

### 2. コーディング時

- 正確な数値（px単位、小数点含む）を参照
- テキスト内容をコピー
- レイアウトモード（row/column）を確認

### 3. 修正時（キャッシュファースト）

**修正・調整セッションでは、MCPログを最優先で参照する。**

```
手順:
1. docs/mcp-log/{page}/ 内のログファイルを開く
2. 必要な値（HEXコード、px値、レイアウト等）を確認
3. ログに存在する → その値を使用（Figma MCP呼び出し不要）
4. ログに存在しない → Figma MCPで取得し、ログに追記
```

**重要**: 初回コーディング時に取得済みの情報をFigma MCPから再取得すると、
同じ情報の二重取得でトークンを大量浪費する。
MCPログは正確なHEXコード・px値で記録されているため、修正時はこれで十分。
