# Figma → HTML/SCSS 自動コーディング システム

FigmaデザインカンプをAI（Claude Code）で自動的にHTML/SCSSに変換するシステムです。
Figma URLを渡すだけで、デザイン情報の取得からコーディング、セルフレビューまでを自動実行します。

この `docs` フォルダ、`.claude/` フォルダ、`CLAUDE.md` をあなたのプロジェクトにコピーして使います。
コーディング環境（ビルドツール、ディレクトリ構成など）はあなたのプロジェクトに合わせてください。

## 前提条件

| 項目 | 必須/推奨 | 備考 |
|------|-----------|------|
| [Claude Code](https://claude.com/claude-code) | 必須 | CLI版のClaude（**MAXプラン推奨**） |
| [Figma MCP](https://github.com/anthropics/figma-mcp) | 必須 | Claude CodeにFigma MCPサーバーを接続 |
| Node.js | 必須 | v18以上推奨 |
| Figma アクセストークン | 必須 | 画像ダウンロードに使用（REST API用） |
| Figma デザインファイル | 必須 | 対象のデザインカンプ |

> **なぜMAXプラン（$100/月）？**: このシステムはFigmaデータ取得・レイヤー分析・コーディング・レビューを自動で連続実行するため、大量のトークンを消費します。$20のProプランではレート制限にすぐ到達し、パイプラインが途中で止まります。MAXプランは月$100ですが、コーダー1人分の工数を自動化できる事業投資と考えてください。

### 対象サイトの適正範囲

| 対象 | 適合度 |
|------|--------|
| コーポレートサイト | OK |
| ランディングページ | OK |
| ブログ・コラムページ | OK |
| position:absolute多用サイト | NG |
| 複雑なアニメーションサイト | NG |

## セットアップ

### 1. docs フォルダ、.claude/ フォルダ、CLAUDE.md をコピー

以下の3つをあなたのプロジェクトのルートにコピーしてください。

```
your-project/
├── CLAUDE.md      ← プロジェクトルール（AIの動作制御に必要）
├── .claude/
│   ├── settings.json  ← hook設定（コンテキスト圧縮対策に必要）
│   └── skills/        ← AIスキル定義（パイプラインの自動実行に必要）
├── docs/              ← パイプライン・規約・パターン等
├── src/
├── package.json
└── ...
```

> **既にこれらのファイルがある場合:**
> - `CLAUDE.md` → 既存の内容を残しつつ、本システムのルールを追記（マージ）してください
> - `.claude/settings.json` → 既存の設定を残しつつ、`PreCompact` hookを追記（マージ）してください
> - `.claude/skills/` → そのままコピーでOK（プロジェクト単位なので他のスキルと競合しません）

> **`.claude/skills/` とは？**
> Claude Codeが各工程で自動的に使うスキル（スラッシュコマンド）の定義ファイルです。
> レイヤー分析、実装計画、セルフレビュー、ピクセル検証などの工程を自動化します。
> Markdownファイルなので、プロジェクトに配置するだけでClaude Codeが自動認識します。
>
> **`.claude/settings.json` とは？**
> Claude Codeのコンテキスト自動圧縮時に、パイプラインのルールが失われるのを防ぐhook設定です。
> 長時間の作業でもAIがルールを忘れず正確にコーディングし続けるために必要です。

含まれるスキル：

| スキル | 概要 | 発動タイミング |
|--------|------|----------------|
| scan | Figmaレイヤー分析 | 工程2で自動 |
| blueprint | 実装計画の作成 | 工程3で自動 |
| mcp-check | MCPログの検証 | 工程4着手前に自動 |
| pattern | パターンテンプレート展開 | 工程3で使用 |
| review | セルフレビュー | 工程5で自動 |
| pixeldiff | ピクセルパーフェクト検証 | オンデマンド |
| rca | 差分の根本原因分析 | pixeldiff後に使用 |
| learn | ナレッジ手動記録 | `/learn [内容]` で手動 |
| auto-learn | ナレッジ自動記録 | 作業中に自動 |

### 2. Figma アクセストークンの設定

`docs/.env` ファイルを作成し、Figmaのアクセストークンを記入してください。

```
FIGMA_ACCESS_TOKEN=your_figma_access_token_here
```

> **アクセストークンの取得方法**: Figma > 設定 > Account > Personal access tokens で生成できます。

### 3. Figma MCPの接続確認

Claude CodeにFigma MCPサーバーが接続されていることを確認してください。
接続されていない状態ではコーディングを開始できません。

### 4. 開発サーバーの起動

AIがスクリーンショットを撮影して視覚チェックを行うため、開発サーバーを起動しておいてください。
コマンドはあなたのプロジェクトの環境に合わせてください（`npm run dev`, `npx vite` など）。

### 5. パーミッション設定（推奨）

Claude Codeはデフォルトでファイル編集やコマンド実行のたびに許可を求めます。
パイプラインを自動で通すには、許可確認をスキップするモードでの起動を**推奨**します。

**起動コマンド：**

```bash
claude --dangerously-skip-permissions
```

さらに、起動時の確認ダイアログもスキップするには、`~/.claude/settings.json` に以下を追加してください：

```json
{
  "skipDangerousModePermissionPrompt": true
}
```

> **なぜ必要？**: パイプラインは工程0〜5を自動実行するため、途中で許可ダイアログが出ると流れが止まります。

### 6. Cursorでの起動方法

Cursor（VS Code）でClaude Codeを使う手順です。

1. `` Cmd + ` ``（バッククォート）でターミナルを開く
2. ターミナルで `claude --dangerously-skip-permissions` を実行
3. 下のターミナルタブを**エディタエリアにドラッグ&ドロップ**すると、コードと並べて表示できます

> **Tips**: `Cmd + Shift + P` → `Terminal: Move Terminal into Editor Area` でも同じことができます。戻すときは `Terminal: Move Terminal into Panel` です。

## カスタマイズ（初回のみ）

**このシステムに含まれる `CODING_RULES.md` と `patterns/` は、作者のコーディングスタイルに最適化されています。**
そのまま使うとあなたのコードの書き方と合わない可能性があるため、初回に必ずカスタマイズしてください。

### コーディング規約のカスタマイズ

`CODING_RULES.md` をあなたのコーディングスタイルに合わせて書き換えます。

**やり方**: あなたの過去のプロジェクトのソースコード（HTML/SCSS/CSS）をプロジェクトに置いた状態で、Claude Codeに以下のように指示してください。

```
プロジェクト内のソースコードを分析して、私のコーディングスタイルに合わせて
docs/CODING_RULES.md を書き換えてください。
命名規則、SCSS記法、HTML構造、クラス設計手法などを読み取って反映してください。
```

### パターンライブラリのカスタマイズ

`patterns/` フォルダにはUIパターンのテンプレート（ヘッダー、カードグリッド、CTAなど）が入っています。
これもあなたのコーディングスタイルに合わせて書き換えます。

```
プロジェクト内のソースコードを分析して、私のコーディングスタイルに合わせて
docs/patterns/ 内のパターンテンプレートを書き換えてください。
HTML構造、クラス命名、SCSSの書き方を私のスタイルに揃えてください。
```

> **ポイント**: 過去のプロジェクトのコードが多いほど、AIが正確にスタイルを読み取れます。
> カスタマイズ後は、過去コードをプロジェクトから削除して問題ありません。

### agent.md のリセット

`agent.md` は作者の作業中に蓄積されたナレッジです。
中身を確認して不要なら空にして構いません。使っていくうちにあなた専用のナレッジが自動で蓄積されていきます。

## 使い方

### 基本の流れ

```
1. 指示書（mdファイル）を書く
2. Claude Codeに指示書を渡す
3. AIが自動でコーディング（工程0〜5）
4. 完了報告を受け取る
5. （必要に応じて）ピクセルパーフェクト検証を依頼
```

### 最もシンプルな使い方

Claude Codeに以下のように指示するだけです：

```
Figma URLとPipelineを渡すので、コーディングをお願いします。

@https://www.figma.com/design/xxxxx/FileName?node-id=XX-XXXX&m=dev
@https://www.figma.com/design/xxxxx/FileName?node-id=XX-XXXX&m=dev
```

AIが `PIPELINE.md` を読み込み、工程0〜5を自動で実行します。

## 指示書の書き方

指示書は `docs/` 直下に `{プロジェクト名}_instruction.md` として作成します。

### 指示書のテンプレート

```markdown
{ページ名}のコーディングをお願い

@https://www.figma.com/design/xxxxx/FileName?node-id=XX-XXXX&m=dev
@https://www.figma.com/design/xxxxx/FileName?node-id=XX-XXXX&m=dev
```

### Figma URLの取得方法

1. Figmaでデザインファイルを開く
2. 対象のフレーム/セクションを選択
3. 右クリック > 「Copy/Paste as」 > 「Copy link」でURLを取得
4. `?node-id=XX-XXXX` 部分がセクションを特定するノードID

### 指示書を分割する場合

セクション数が多い場合は、指示書を分割してください（例: `_instruction_01.md`, `_instruction_02.md`）。
分割した場合は、Claude Codeにファイルを1つずつ渡して順に実行します。

```
docs/
├── mysite_instruction.md      ← 初回セッション用（変数・骨格の構築）
├── mysite_instruction_01.md   ← セクション1〜3
├── mysite_instruction_02.md   ← セクション4〜6
└── mysite_instruction_03.md   ← セクション7〜9
```

## 注意点・Tips

### 1. Figma URLの数は1回の指示で3〜4個まで

**1回の指示にFigma URLを詰め込みすぎると、AIの精度が落ちます。**

URLが多いとコンテキストウィンドウを圧迫し、デザイン情報の取りこぼしやコーディングミスが増えます。

- 1回の指示で渡すセクション数: **1〜2個**が推奨
- セクション数が多い場合: **指示書を分割**する
- 分割の判断が難しい場合: AIに「この指示書を適切な粒度に分割してください」と依頼する

### 2. 複雑なSVGは手動でエクスポートが必要

マスクや複数レイヤーで構成されたSVGイラスト（人物、キャラクター、装飾的なイラストなど）は、AIが自動でダウンロードできません。

AIがコーディング完了時に以下のように報告します：

```
以下のSVGはFigmaから手動エクスポートをお願いします:
- イラスト名 (node-id: XX-XXXX) → src/img/{page}/{filename}.svg
```

**手動エクスポートの手順：**
1. Figmaで該当ノードを選択
2. 右パネルの「Export」でSVG形式を選択
3. エクスポートしたファイルを指定パスに保存
4. AIに「SVGを配置しました」と伝えると、コードに組み込まれます

### 3. まとめて書き出したい画像はFigmaでグループ化して「png」と命名

複数レイヤーで構成された画像（写真+オーバーレイ、装飾付きイラストなど）を1枚の画像として書き出したい場合は、Figma上で対象レイヤーをグループ化し、グループ名に `png` を含めてください（例: `hero-png`, `bg-png`）。

AIはグループ名に `png` が含まれるレイヤーをPNG画像として一括ダウンロードします。

### 4. .envファイルのアクセストークンは必須

画像のダウンロードにFigma REST APIを使用するため、`docs/.env` にアクセストークンが必要です。
トークンがないと画像が取得できず、コーディングが途中で止まります。

### 5. 開発サーバーは事前に起動しておく

AIがスクリーンショットを撮影して視覚チェックを行うため、開発サーバーが起動している必要があります。

### 6. PCのみ / PC+SP の指示

| 提供するURL | 動作 |
|---|---|
| PCのFigma URLのみ | PC実装 + 基本レスポンシブ（SPは経験則ベース） |
| PC + SP両方のURL | PC実装 + Figma準拠のSP実装（ピクセルパーフェクト検証可能） |

SP（スマホ）のFigma URLも渡すとより正確なレスポンシブ対応が可能です。

## パイプライン概要

AIが以下の工程を自動で実行します（工程0〜5は自動、工程6〜7はオンデマンド）。

```
工程0 デザインシステム構築  ← Figmaからカラー・フォント変数を取得
  ↓
工程1 画像エクスポート      ← REST APIで画像ダウンロード
  ↓
工程2 Figmaレイヤー分析    ← レイヤー構造をHTML/CSSにマッピング
  ↓
工程3 実装計画              ← パターン選定・HTML/SCSS方針を決定
  ↓
工程4 コーディング          ← 計画に基づいてHTML→SCSSを実装
  ↓
工程5 セルフレビュー        ← 規約照合 + 視覚チェック
  ↓
  ✅ コーディング完了 → ユーザーに報告
  ↓（ユーザーが依頼した場合のみ）
工程6 ピクセルパーフェクト検証
  ↓
工程7 品質チェック
```

各工程の詳細は [PIPELINE.md](./PIPELINE.md) を参照してください。
各工程は `.claude/skills/` に定義されたスキルによって自動実行されます。

## トラブルシューティング

### 「Figma MCPが接続されていません」と表示される

Claude CodeにFigma MCPサーバーが接続されていません。Claude Codeの設定でFigma MCPを追加してください。

### 画像がダウンロードされない

`docs/.env` にFigmaアクセストークンが正しく設定されているか確認してください。トークンの有効期限が切れている場合は、Figmaで再発行してください。

### コーディング精度が低い

- 1回の指示でFigma URLを詰め込みすぎていないか確認（3〜4個まで推奨）
- 指示書を分割して、セッションを分けて実行してみてください
- `CODING_RULES.md` と `patterns/` をあなたのスタイルにカスタマイズしたか確認してください

### SVG画像が表示されない

複雑なSVG（マスク・複数レイヤー構成）はAIが自動取得できません。完了報告で案内されたノードIDをFigmaで選択し、手動でSVGエクスポートしてください。
