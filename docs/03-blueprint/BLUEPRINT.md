# 工程3: 実装計画

## 概要

各セクションの実装計画をコード着手前に作成する。
Figmaレイヤー分析（工程2）の結果をもとに、HTML構造・SCSS方針・画像取得計画を事前に決定する。
この工程はAIが自動進行する（ユーザーの承認不要）。

---

## 目的

- コーディング前にHTML構造とSCSS方針を確定させる
- パターンライブラリから適用パターンを選定し、テンプレートを活用する
- 過去の実装例を参照して同じミスを避ける
- リスク箇所を事前に特定し、対処方針を決める
- 計画書を `docs/mcp-log/{page}/{section}-plan.md` に保存する

---

## 手順

### 1. パターンの選定

[パターンライブラリ](../patterns/README.md) から、対象セクションに適用するパターンを選定する。

**選定基準:**
- 工程2のレイヤー分析結果とパターンの「Figmaでの特徴」を照合
- 複数パターンの組み合わせも可（例: パターン03+04+10）
- 完全一致しなくても、最も近いパターンをベースにカスタマイズ

### 2. 類似実装例の参照

[実装例ディレクトリ](../examples/README.md) から、類似セクションの実装方法を参照する。

- HTML構造のクラス命名パターンを確認
- SCSS方針（flex/grid、レスポンシブ戦略）を参考にする
- 意思決定の記録から、過去の判断理由を理解する

### 3. HTML構造の設計

以下を事前に決定する:
- セクション全体の構造（ネスト図）
- 各要素のHTMLタグ（div, section, figure, span, h2, h3等）
- クラス名（FLOCSSのp-{section}__{element}形式）
- セマンティックHTML要素の使用箇所

### 4. SCSS方針の決定

以下を事前に決定する:
- レイアウト手法（flex / grid / 通常フロー）
- レスポンシブ戦略（どのブレークポイントで何を変えるか）
- 背景の実装方法（CSS / 1枚画像書き出し）
- 特殊なプロパティ（transform, mix-blend-mode等）の使用箇所

### 5. 画像リストの確定

工程2のレイヤー分析で特定した画像を整理する:
- ファイル名の命名（命名規則に従う）
- 取得方法（REST API / アセットURL）
- 形式（SVG / PNG / JPG）
- 保存先パス

### 6. リスク箇所の対処方針

以下に該当する箇所の対処方針を決定する:
- CSS再現が困難な装飾（→ 画像書き出し）
- テキストの部分スタイル変更（太字、色違い → span分離）
- レスポンシブで崩れやすい箇所（→ ブレークポイント戦略）
- position: absolute が必要に見える箇所（→ 代替手法検討）

### 7. 計画書の保存

[SECTION_PLAN_TEMPLATE.md](./SECTION_PLAN_TEMPLATE.md) のテンプレートを使用して計画書を作成し、
`docs/mcp-log/{page}/{section}-plan.md` に保存する。

---

## チェックポイント

計画書作成後、以下を確認してから工程4に進む:

- [ ] 適用パターンが選定されている
- [ ] HTML構造のネスト図が作成されている
- [ ] 全要素のクラス名が決定されている
- [ ] SCSS方針（レイアウト手法、レスポンシブ）が決定されている
- [ ] 画像リストが確定している
- [ ] リスク箇所の対処方針が決定されている
